[题目传送门](https://www.luogu.com.cn/problem/P4170)

区间 dp 板子题。

设 $dp[i][j]$ 为左端点为 $i$，右端点为 $j$ 的区间染成给定颜色的最少步数，显然 $dp[i][i]=1$。

然后枚举一下断点 $k$，就有 $dp[i][j]=\min(dp[i][j],dp[i][k]+dp[k+1][j])$。

但是这样是不行的，考虑如果左端点 $i$ 和右端点 $j$ 同色，因为染一个长区间的操作数严格不劣于染两个短区间的操作数（这个结论十分直观），那么只需要从之前的某个染色方案中多染一个就行了。

因此这部分转移方程就是 $dp[i][j]=\min(dp[i][j-1],dp[i+1][j])$。（即取扩展左端点和扩展右端点的方案最小值）


注意第一重循环要枚举区间长度！！！

时间复杂度 $O(n^3)$。

代码：
```cpp
#include<bits/stdc++.h>
#define ri register int
#define back return
#define ll long long
using namespace std;
char a[105];
int dp[105][105];
int main()
{
	cin>>a+1;
	int n=strlen(a+1);
	for(ri i=1;i<=n;i++)
		for(ri j=1;j<=n;j++)
			if(i==j)
				dp[i][j]=1;
			else
				dp[i][j]=1e9;
	for(ri len=2;len<=n;len++)
		for(ri i=1;i<=n;i++)
		{
			int j=i+len-1;
			if(a[i]==a[j])
				dp[i][j]=min(dp[i][j-1],dp[i+1][j]);
			else
				for(ri k=i;k<j;k++)
					dp[i][j]=min(dp[i][j],dp[i][k]+dp[k+1][j]);
		}
			
	cout<<dp[1][n]<<"\n";
	back 0;
}
```
