## 一、一些常识

SPFA 求最短路，本质上是队列优化的 Bellman-Ford。

Bellman-Ford 的松弛操作就是枚举每个点 $i$ 的所有出边 $(i,j)$，然后看一下 $i,j$ 是否满足三角形不等式 $d_j<d_i+w(i,j)$，不满足就更新 $d_j$。然后当整个图都不能再松弛的时候就结束。

SPFA 就是上述做法的队列优化版本。用 $vis_i$ 代表 $i$ 当前是否在队列中。然后每次取出队头进行松弛操作，当队列为空时结束。

曾经有一篇论文说这个 SPFA 是 $O(km)$ 的……但是显然这个东西寄了。

然后其实我没学过 Bellman-Ford……所以这个东西的其他优化也写不了。

常见的优化有：

堆优化，允许一个点入队多次，然后显然有负权就很可能会被卡成指数级，直接寄了。

栈优化，用栈代替队列。这样 bfs 就变成了 dfs。栈后进先出的特性，在找负环的时候比较有用（可能会一直找环上的点直到把整个环遍历一遍，不浪费时间），但是显然正常图上就可能被卡成指数级，直接寄了。

LLL 优化，双端队列维护，每次比较入队点距离和队内点距离平均值，较大则插入队尾，反之队首。

SLF 优化，双端队列维护，每次比较入队点距离和队首点距离，较大则插入队尾，反之队首。

这两个据说比较有用的优化我不太会，所以略过。但是卡这个东西好像已经是 [出题人必备知识点](https://www.zhihu.com/question/292283275/answer/484871888)。

## 二、其他应用

SPFA 判负环效率很高。

Bellman-Ford 判负环，是看松弛操作进行了多少轮，如果松弛操作超过了 $n-1$ 轮，即第 $n$ 次循环的时候依然在松弛，说明有个负环。

显然这个会被卡满，效率很低。

SPFA 判负环更加方便。记录一个 $cnt_i$ 表示 $i$ 点的最短路经过了 $cnt_i$ 条边。然后假如这个图有 $n$ 个点，那么 $cnt_i \ge n$ 就代表在某一条边上跑了不止一遍，那么一定有负环。
 
每次更新距离的时候就 $cnt_y=cnt_x+1$ 即可，这样复杂度就跟 SPFA 一样。

当然还是会被卡。但是不刻意构造数据的话，这样做差不多比无优化 Bellman-Ford 快好几倍。

注意判负环的时候没有无解，只能证明从起点出发不能到达负环，而不是图上没有负环。

所以判断全图是否有负环时，常用的方法是建一个超级原点，然后从这个点向每个节点连一条权值为 $0$ 的边，然后从这个超级原点出发跑最短路。

```cpp
//一个板子
void spfa(int s)
{
	q.push(s),dis[s]=0,cnt[0]=0;
	while(!q.empty()&&flag)
	{
		int now=q.front();
		q.pop();
		vis[now]=0;
		for(ri i=0;i<a[now].size();i++)
		{
			int to=a[now][i],d=w[now][i];
			if(dis[to]>dis[now]+d)
			{
				dis[to]=dis[now]+d;
				cnt[to]=cnt[now]+1;
				if(cnt[to]>=n+1)
				{
					flag=0;
					break;
				}
				if(!vis[to])
					q.push(to),vis[to]=1;
			}
		}
	}
}
```
